## Phase 2: Engagement & AI Security (Current)

We are currently operating in Phase 2. While the core prediction mechanics are solid, the platform requires extensive features to drive user engagement, retain liquidity, and ensure flawless performance under heavy load.

<Callout type="tip" title="DotShield AI Active">
  The Arc Testnet is now actively monitored by **DotShield**, our autonomous self-healing AI Guardian engine. DotShield continuously assesses RPC node health, Pyth price feed freshness, and keeper heartbeats every 5 seconds to perform automatic failovers and guarantee 100% uptime.
</Callout>

### Current Development Priorities

- **DotShield Embedded Guardian:** Enhancing the self-healing scripts to carry out automatic keeper process restarts and RPC node rotations during network congestion or rate-limiting events.
- **Gas Optimization:** Rewriting specific state variables within the Solidity contracts to drastically reduce the gas required to execute a prediction.
- **Dynamic Payout Display:** Overhauling the frontend logic to calculate and display the real-time payout multipliers faster as pools fluctuate.
- **Global Leaderboard:** Implementing an on-chain tracking system that ranks users based on win rate, volume, and winning streaks.

<Callout type="warning" title="Testnet Volatility">
  During Phase 2, the Arc Testnet contracts may be periodically paused or upgraded without prior warning. This is necessary to ensure we can hot-fix bugs and implement optimizations rapidly.
</Callout>

## Phase 3: Mainnet & Ecosystem (Future Vision)

Phase 3 represents the culmination of our testing. Once the protocol has operated flawlessly on the Arc Testnet with thousands of concurrent users under DotShield supervision, we will prepare for real economic value.

### Upcoming Milestones

#### 1. Security Audits & Mainnet Launch
Before transitioning from the Arc Testnet, the entire codebase will be frozen and submitted to multiple tier-1 blockchain security firms. Only upon receiving pristine audit reports will DotMarket be deployed to a high-throughput Mainnet.

#### 2. DotShield Autonomous Governance
Upgrade DotShield to handle autonomous risk parameter adjustments on mainnet. The AI Guardian will dynamically throttle round execution times or increase pool bounds during high-slippage market fluctuations to shield user liquidity.

#### 3. Multi-Asset Expansion
Following mainnet stabilization, we will implement governance votes to introduce new prediction pools for assets like Ethereum (ETH) and Solana (SOL).

#### 4. Algorithmic Trading API
To cater to our most sophisticated users, we will release a comprehensive SDK and WebSocket API. This will allow developers to run algorithmic bots that ingest Oracle data and automatically execute predictions.
